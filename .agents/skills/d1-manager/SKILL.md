# Instructions

1. **Drizzle-First Approach:** Always use Drizzle Kit for managing the schema instead of raw SQL migrations. All schema definitions live in `packages/db/src/schema/`.
2. **Generating Migrations:** After altering the schema, generate new migration files by running `bun run --filter @cornwall-ponds-gemini/db db:generate`.
3. **Pushing Changes:** Apply the schema changes to your database by running `bun run --filter @cornwall-ponds-gemini/db db:push`. Note: Drizzle's D1-HTTP driver requires variables like `CLOUDFLARE_ACCOUNT_ID`, `CLOUDFLARE_DATABASE_ID`, and `CLOUDFLARE_D1_TOKEN` to push remotely.
4. **Local D1 execution:** If you absolutely need to run RAW sql on the local Cloudflare instance, use `bunx wrangler d1 execute <YOUR_DB_NAME> --local --file=<FILE_PATH>`. Always run this with `--dry-run` first to verify the output.
5. **No Manual Migration Edits:** Do not manually edit the SQL files generated by Drizzle inside `packages/db/src/migrations/`. Let Drizzle manage them.
