BeforeAfter.tsx is the container/section component. It owns the list of projects and the currentIndex state, selects the active project, and renders the whole section: heading copy, instructions, slider, navigation, dots, and the CTA. It now imports the three helpers via ./before-and-after/..., so its main responsibility is orchestration: it wires project data and callbacks into the more focused subcomponents.

BeforeAfterSlider.tsx contains the visual before/after comparison widget only. It receives beforeImage, afterImage, and title, manages slider drag state (sliderValue) and image error fallbacks internally, and renders the overlapping images, clipped “after” image, divider line, drag handle, and invisible range input. This file is where you’d tweak the look-and-feel or behavior of the comparison effect without touching project navigation or section layout.

BeforeAfterProjectNav.tsx is responsible for the project navigation bar under the slider. It takes title, location, hasMultipleProjects, and onPrev/onNext handlers. It renders the prev/next buttons (only if there are multiple projects) and the centered title + location. If you want to change labels, icons, or accessibility attributes for project navigation, this is the place to do it.

BeforeAfterDots.tsx handles the dot/tab indicator strip. It receives a minimal projects list ({ title, slug }), the currentIndex, and an onSelect(index) callback. It renders the dots with the correct active styling and ARIA roles, and calls back to BeforeAfter.tsx when a user chooses a specific project. That keeps the selection state in one place (the container) while giving the dots a very small, testable surface.
